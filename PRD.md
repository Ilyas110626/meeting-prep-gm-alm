| Поле | Значение |

---

## 1. TL;DR

Персональный инструмент, который ведёт список предстоящих встреч, напоминает о них с обратным отсчётом и для каждой встречи формирует конкретный план подготовки аналитического материала, привязанный к направлению **Global Markets (GM)** или **Asset & Liabilities Management (ALM)**.

Продукт включает два режима, построенных на единой логике и дизайне:

- **Режим A — Трекер встреч.** Универсальный список встреч с авто-генерацией плана подготовки по ключевым словам темы.
- **Режим B — Бриф к встрече с контрагентом.** Готовый бриф к конкретной встрече по фиксированной повестке (FX и локальный рынок, ставки и курс, ликвидность и первичное дилерство, Euroclear, индекс JP Morgan), с возможными вопросами контрагента, разнесёнными по линиям GM и ALM.

---

## 2. Проблема и контекст

Специалист GM/ALM регулярно участвует во встречах (внутренние комитеты, ALCO, переговоры с банками-контрагентами), к каждой из которых нужно приходить с актуальной рыночной картиной, выверенными цифрами и обоснованной позицией. Подготовка сегодня:

- разрознена — данные собираются вручную из разных источников каждый раз;
- неструктурирована — нет единого шаблона «что именно подготовить» под тип встречи;
- реактивна — встречные вопросы контрагента часто прорабатываются «на ходу».

**Следствие:** время на подготовку растёт, качество материала зависит от загрузки, риск прийти неподготовленным к острым вопросам.

---

## 3. Цели и не-цели

### 3.1 Цели

1. Сократить время на подготовку аналитического материала к встрече.
2. Стандартизировать структуру подготовки по направлениям GM и ALM.
3. Заранее проработать возможные вопросы контрагента и тезисы-ответы.
4. Обеспечить своевременное напоминание о встрече с обратным отсчётом.
5. Дать материал в формате, готовом к копированию в заметки/письмо.

### 3.2 Не-цели (вне рамок продукта)

- Не является торговой/расчётной системой и не исполняет сделок.
- Не даёт персональные инвестиционные рекомендации.
- Не заменяет первоисточники данных — цифры являются ориентирами и требуют обновления перед встречей.
- Не предоставляет серверных push-уведомлений (напоминания работают, пока приложение открыто).
- Не является многопользовательской платформой (продукт для индивидуального использования).

---

## 4. Целевые пользователи и сценарии

**Основной пользователь:** специалист казначейства / Global Markets / ALM в банке, готовящийся к встречам и комитетам.

| Сценарий | Описание | Режим |
|---|---|---|
| Подготовка к ALCO | Нужен план по ликвидности, IRRBB, фондированию | A |
| Подготовка к рыночному обзору | Нужен план по FX / ставкам / кредиту | A |
| Встреча с банком-контрагентом | Нужен бриф по фиксированной повестке + ожидаемые вопросы | B |
| Быстрое напоминание | «Что у меня завтра и насколько я готов» | A / B |

## 6. Функциональные требования

### 6.1 Управление встречами (Режим A) — [MVP]

| ID | Требование |
|---|---|
| F-A1 | Пользователь может добавить встречу: тема, дата, время, направление (GM/ALM), участники, заметки. |
| F-A2 | Список встреч отсортирован по времени по возрастанию. |
| F-A3 | Фильтрация списка по направлению: Все / Global Markets / ALM, со счётчиком в каждой вкладке. |
| F-A4 | Удаление встречи. |
| F-A5 | При первом запуске показываются два примера (GM и ALM), помеченные «(пример)», которые можно удалить. |

### 6.2 Напоминания и обратный отсчёт — [MVP]

| ID | Требование |
|---|---|
| F-R1 | Для каждой встречи отображается обратный отсчёт: «через N мин», «сегодня · через N ч M мин», «завтра в HH:MM», «через N дн», «идёт сейчас», «завершена». |
| F-R2 | Цвет статуса меняется по срочности: <1 ч — критичный (красный), сегодня — предупреждающий (янтарный), позже — нейтральный. |
| F-R3 | Обратный отсчёт пересчитывается автоматически (раз в минуту) без перезагрузки. |
| F-R4 | В шапке отображаются текущие дата и время (часы). |

### 6.3 Генерация плана подготовки (Режим A) — [MVP]

| ID | Требование |
|---|---|
| F-P1 | Для каждой встречи генерируется структурированный план из пяти блоков: (0) контекст и цель, (1) ключевые данные к сбору, (2) материалы к подготовке, (3) сценарии и вопросы, (4) рекомендуемая позиция/итог. |
| F-P2 | План адаптируется к теме: по ключевым словам в теме и заметках распознаются тематические модули и подставляется релевантная конкретика направления. |
| F-P3 | При отсутствии совпадений по ключевым словам используется базовый план направления с пометкой «общий план направления». |
| F-P4 | План можно скопировать в буфер обмена как форматированный текст. |

### 6.4 Бриф к встрече с контрагентом (Режим B) — [MVP]

| ID | Требование |
|---|---|
| F-B1 | Шапка-бриф содержит контрагента, дату/время, участников и обратный отсчёт-напоминание. |
| F-B2 | Детали встречи (контрагент, дата, время, участники) редактируются и сохраняются локально. |
| F-B3 | Повестка из 5 фиксированных тем отображается раскрывающимися карточками, пронумерованными 01–05. |
| F-B4 | Каждая тема помечена линией/линиями: Global Markets и/или ALM. |
| F-B5 | Каждая тема содержит блоки: наша позиция и тезисы, ключевые цифры (плашки), возможные вопросы контрагента по линии GM, возможные вопросы по линии ALM, рекомендуемый тезис в ответ. |
| F-B6 | Возможные вопросы контрагента визуально разделены по линиям GM (терракота) и ALM (бирюза). |
| F-B7 | Фильтр: Все темы / Вопросы Global Markets / Вопросы ALM — позволяет сфокусироваться на одной линии. |
| F-B8 | Копирование отдельной темы и копирование всего брифа в буфер обмена. |
| F-B9 | Над повесткой отображается баннер-предупреждение, что цифры — ориентиры на дату источников и требуют обновления перед встречей. |
| F-B10 | Внизу перечислены источники для проверки данных. |

### 6.5 Контент повестки Режима B (v1.0)

| № | Тема | Линии |
|---|---|---|
| 01 | Взгляд на FX и динамику локального рынка | GM, ALM |
| 02 | Ожидания по ставке и обменному курсу | GM, ALM |
| 03 | Ликвидность и система первичного дилерства | ALM, GM |
| 04 | Предстоящее внедрение Euroclear | GM, ALM |
| 05 | Возможное включение в индекс локальных облигаций JP Morgan | GM, ALM |

## 8. Пользовательские потоки

### 8.1 Режим A — добавление встречи и подготовка
1. Пользователь нажимает «Встреча» → заполняет тему, дату, время, направление, заметки → «Сохранить».
2. Встреча появляется в списке с обратным отсчётом.
3. Пользователь раскрывает карточку → видит сгенерированный план из 5 блоков.
4. При необходимости копирует план в буфер.

### 8.2 Режим B — бриф к встрече с контрагентом
1. Пользователь открывает бриф → видит шапку-напоминание и повестку.
2. Нажимает «Изменить детали встречи» → задаёт контрагента, дату, время, участников → отсчёт пересчитывается.
3. Раскрывает темы, изучает позицию, цифры и возможные вопросы по линиям GM/ALM.
4. Переключает фильтр на нужную линию для фокусной проработки вопросов.
5. Копирует тему или весь бриф; перед встречей обновляет рыночные цифры по источникам.

## 9. Модель данных

### 9.1 Встреча (Режим A)
```
Meeting {
  id: string            // уникальный идентификатор
  title: string         // тема встречи
  div: 'gm' | 'alm'     // направление
  dt: string            // дата-время (локальный ISO, "YYYY-MM-DDTHH:mm")
  att: string           // участники / контрагент (необязательно)
  notes: string         // заметки (уточняют генерацию плана)
}
```

### 9.2 Деталь встречи с контрагентом (Режим B)
```
Meeting {
  cp: string            // банк-контрагент
  dt: string            // дата-время
  att: string           // участники со стороны контрагента
}
```

### 9.3 Тема повестки (Режим B, контент)
```
Topic {
  id, num, title, en          // идентификатор, номер, название, англ. формулировка
  lines: ['gm' | 'alm']       // линии направления
  position: string[]          // позиция и тезисы
  figures: {k, v}[]           // ключевые цифры (метка/значение)
  qGM: string[]               // вопросы контрагента — линия Global Markets
  qALM: string[]              // вопросы контрагента — линия ALM
  rec: string                 // рекомендуемый тезис в ответ
}
```

## 10. Контент-логика (база знаний и генерация)

### 10.1 Базовые чек-листы направлений (Режим A)
- **Global Markets:** снимок рынка на дату, календарь рыночных событий, позиционирование и клиентские потоки, риск-метрики и P&L.
- **ALM:** структура баланса, нормативы ликвидности (LCR/NSFR), профиль процентного риска (IRRBB: EVE/NII), структура и стоимость фондирования (FTP).

### 10.2 Тематические модули (распознаются по ключевым словам)
- **GM:** FX, ставки/облигации, кредит, акции, товары, волатильность.
- **ALM:** ликвидность, IRRBB, фондирование, депозиты, капитал, баланс.

Для каждой темы детектируются один или несколько модулей; их конкретные пункты данных добавляются к базовому блоку направления.

### 10.3 Источники данных для контента Режима B (актуализация)
- **Национальный Банк РК** — базовая ставка, официальный курс, инфляция, инструменты ликвидности.
- **Минфин РК** — план первичного дилерства, календарь и параметры эмиссии.
- **Euroclear / KCSD** — статус расчётной инфраструктуры (direct link, DVP).
- **J.P. Morgan Index Research** — методология и критерии GBI-EM.
- **ЕАБР, S&P** — прогнозы по курсу и ставке.

> **Важно:** числовые значения в продукте — справочные ориентиры на дату источников. Перед каждой встречей обязательна актуализация рыночных уровней (USD/KZT, доходности ГЦБ, базовая ставка, последние решения регулятора).

## 11. UX и дизайн-принципы

- **Визуальная система:** институциональный «казначейский» стиль; два цвета-кода направлений (GM — терракота, ALM — бирюза) как смысловая, а не декоративная маркировка.
- **Типографика:** дисплейный гротеск для заголовков, нейтральный sans для текста, моноширинный шрифт для данных и обратного отсчёта.
- **Структура:** раскрывающиеся карточки, нумерованные блоки плана, цветовой «корешок» по направлению.
- **Тон копирайтинга:** деловой, без украшательств; формулировки описывают действия пользователя.
- **Доступность:** мобильная адаптивность, видимый фокус, уважение к reduced-motion.

## 12. Технические решения / архитектура

| Аспект | Решение |
|---|---|
| Формат поставки | Один HTML-файл (HTML + CSS + JS), без сборки. |
| Зависимости | Только веб-шрифты (Google Fonts); вся логика — нативный JS. |
| Хранение | Локальное персистентное хранилище приложения (ключ-значение), с graceful-fallback в память. |
| Состояние | В оперативной памяти на время сессии; персист — детали встреч/список. |
| Обновления | Раз в минуту — пересчёт обратного отсчёта; раз в секунду — часы в шапке. |

| Метрика | Целевой ориентир |

## 15. Риски и ограничения

| Риск | Влияние | Митигирование |
|---|---|---|
| Устаревание цифр в контенте | Неверная позиция на встрече | Баннер-предупреждение + регламент актуализации по источникам; в перспективе — авто-подгрузка |
| Напоминания работают только при открытом приложении | Пропуск встречи | В перспективе — серверные/календарные уведомления |
| Локальное хранение — данные привязаны к устройству/браузеру | Потеря данных при смене окружения | В перспективе — экспорт/синхронизация |
| Ключевые слова не покрывают все темы (Режим A) | Слишком общий план | Резервный базовый план + расширение словаря модулей |
